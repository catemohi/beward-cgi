## Общее описание
Управление ключами для панелей Beward

## Доступные команды:
1. `eqmup`: Загрузить ключи на панель из EQM файла
2. `d2j`: Выгрузить ключи с панели в JSON
3. `lj`: Загрузить ключи на панель из JSON
4. `zipup`: Загрузить ключи на панель из ZIP-архива

## Общие опции 

- `-h`, `--help`: Показать справочное сообщение и выйти

---

## Команда `eqmup`

Загружает ключи на панель из EQM файла.

### Подкоманда `host`

Запускает скрипт для одного адреса. Это полезно, когда необходимо работать с одной конкретной панелью.

#### Аргументы:

- `ip`: IP адрес панели (обязательный атрибут).
- `filepath`: Путь к файлу с ключами (обязательный атрибут).
- `-u`, `--username`: Имя пользователя, зарегистрированного на панели Beward.
- `-p`, `--password`: Пароль пользователя, зарегистрированного на панели Beward.
- `-h`, `--help`: Показать это сообщение и выйти.


### Подкоманда `list`

Используется для загрузки ключей на панели из списка адресов, предоставленного в CSV файле. Удобно для работы с несколькими панелями определенного списка.

#### Аргументы:

- `csvpath`: Путь к CSV файлу со списком адресов (обязательный атрибут).
- `filepath`: Путь к файлу с ключами (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `--thread`: Количество потоков для запуска скрипта.
- `-h`, `--help`: Показать это сообщение и выйти.

### Подкоманда `string`

Позволяет загрузить ключи на панели, адреса которых перечислены через запятую в текстовой форме. Это простой способ выполнения задачи без создания внешних файлов.

#### Аргументы:

- `string`: Список IP-адресов, разделенных запятой (обязательный атрибут).
- `filepath`: Путь к файлу с ключами (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `--thread`: Количество потоков для запуска скрипта.
- `-h`, `--help`: Показать это сообщение и выйти.

### Пример использования:
1. Загрузка ключей на панель из EQM файла:
   ```
   python3 keys.py eqmup host 192.168.0.1 <путь/к/файлу>
   ```
   
2. Загрузка ключей на панель из EQM файла с указанием имени пользователя и пароля:
   ```
   python3 keys.py eqmup host 192.168.0.1 <путь/к/файлу> --username admin --password 123456
   ```

3. Загрузка ключей на панель из EQM файла с указанием имени пользователя и пароля:
   ```
   python3 keys.py eqmup host 192.168.0.1 <путь/к/файлу> --username admin --password 123456
   ```

4. Загрузка ключей на панель из EQM файла с указанием имени пользователя и пароля.
   ```
   python3 keys.py eqmup host 192.168.0.1 <путь/к/файлу> --username admin --password 123456
   ```

5. Загрузка ключей на список панелей из строки, ключи из EQM файла с указанием имени пользователя и пароля, в два потока:
    ```
    python module_name.py eqmup string 10.80.1.200,10.80.1.211 <путь/к/файлу>   --username admin --password admin --thread 2 
    ```

6. Загрузка ключей на список панелей из CSV-файла, ключи из EQM файла с указанием имени пользователя и пароля:
    ```
    python module_name.py eqmup list <csvpath> <путь/к/файлу> --username <username> --password <password>
    ```

## Команда `d2j`

Команда `d2j` (dump to JSON) предназначена для выгрузки ключей с панели в файл формата JSON. Эта команда используется когда нужно сделать резервную копию ключей, перенести ключи на другую панель, или просто организовать хранение ключей в структурированном виде.

### Подкоманда `host`

Выполняет экспорт ключей с конкретной панели (хоста) и сохраняет их в JSON файле. Используется для работы с одним устройством.

#### Аргументы:

- `ip`: IP адрес панели (обязательный атрибут).
- `-u`, `--username`: Имя пользователя для панели Beward.
- `-p`, `--password`: Пароль пользователя для панели Beward.
- `--filepath`: Путь для сохранения выгруженного JSON файла (по умолчанию текущая директория).
- `--format_type`: Формат ключей (RFID или MIFARE, по умолчанию "MIFARE")
- `-h`, `--help`: Показать помощь и выйти.


### Подкоманда `list`

Экспортирует ключи с нескольких панелей, перечисленных в CSV файле, и сохраняет их в JSON файлах. Подходит для массовой обработки нескольких устройств.

#### Аргументы:

- `csvpath`: Путь к CSV файлу с перечнем хостов (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `--thread`: Количество потоков для выполнения операции.
- `--filepath`: Путь для сохранения выгруженных JSON файлов (по умолчанию текущая директория).
- `--format_type`: Формат ключей (RFID или MIFARE, по умолчанию "MIFARE").
- `-r`, `--archiveted`: Если указан, то результат будет упакован в ZIP-архив.
- `-h`, `--help`: Показать помощь и выйти.

### Подкоманда `string`

Позволяет экспортировать ключи с панелей, IP-адреса которых заданы в одной строке, разделенные запятыми. Удобно для быстрой выгрузки ключей из предварительно подготовленного списка адресов в текстовом виде.

#### Аргументы:

- `string`: Строка с IP-адресами панелей, разделенными запятыми (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `--thread`: Количество потоков для выполнения операции.
- `--filepath`: Путь для сохранения выгруженных JSON файлов (по умолчанию текущая директория).
- `--format_type`: Формат ключей (RFID или MIFARE, по умолчанию "MIFARE").
- `-r`, `--archiveted`: Если указан, то результат будет упакован в ZIP-архив.
- `-h`, `--help`: Показать помощь и выйти.

### Пример использования:

1. Выгрузка ключей с панели в JSON:
   ```
   python3 keys.py host d2j 192.168.0.1 --filepath path/to/output.json
   ```

2. Выгрузка ключей с панели в JSON с указанием формата ключей:
   ```
   python3 keys.py host d2j 192.168.0.1 --filepath path/to/output.json --format_type RFID
   ```

3. Выгрузка ключей с панели в JSON с указанием имени пользователя и пароля:
   ```
   python3 keys.py host d2j 192.168.0.1 --username admin --password 123456 --filepath path/to/output.json
   ```

4. Выгрузка ключей с панели в JSON с указанием формата ключей и пути сохранения файла:
   ```
   python3 keys.py host d2j 192.168.0.1 --filepath path/to/output.json --format_type MIFARE --username admin --password 123456
   ```

5. Выгрузка ключей с одной панели в JSON файл:
    ```
    python3 keys.py d2j host 192.168.0.1 --filepath path/to/output.json --format_type   MIFARE --username admin --password 123456
    ```

6. Выгрузка ключей со списка панелей из CSV файлa в JSON, с упаковкой в zip-архив:
    ```
    python3 keys.py d2j list path/to/hosts.csv --filepath path/to/output_folder  --format_type MIFARE --username admin --password 123456 --thread 5 --archiveted
    ```

7. Выгрузка ключей с перечисленных в строке панелей в JSON, с заданием количества потоков:
    ```
    python3 keys.py d2j string "192.168.0.1,192.168.0.2" --filepath path/to/    output_folder --format_type MIFARE --username admin --password 123456 --thread 2
    ```

## Команда `lj`

Команда lj (load from JSON) используется для загрузки ключей на панель из файла формата JSON. Это полезно в случаях, когда нужно восстановить резервные копии ключей на панели, настроить новую панель с существующими ключами, или провести массовую регистрацию ключей.

### Подкоманда `host`
Загружает ключи на одну панель из файла JSON. Идеальна для работы с одним устройством, когда нужно восстановить или заменить ключи.

#### Аргументы:

- `ip`: IP адрес панели (обязательный атрибут).
- `filepath`: Путь к JSON файлу ключей (обязательный атрибут).
- `-u`, `--username`: Имя пользователя, зарегистрированного на панели Beward.
- `-p`, `--password`: Пароль пользователя, зарегистрированного на панели Beward.
- `-h`, `--help`: Показать это сообщение и выйти.

### Подкоманда `list`

Загружает ключи на несколько панелей перечисленных в CSV файле из файла JSON. Подходит для одновременной загрузки одинаковых ключей на серию устройств.

#### Аргументы:

- `csvpath`: Путь к CSV файлу со списком адресов (обязательный атрибут).
- `filepath`: Путь к JSON файлу ключей (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `-h`, `--help`: Показать это сообщение и выйти.

### Подкоманда `string`

Позволяет загрузить ключи из файла JSON на панели, адреса которых указаны как строка с разделителями. Это удобно, если нужно быстро выполнить описанную задачу без необходимости создания внешних файлов.

#### Аргументы:

- `string`: Список IP-адресов, разделенных запятой (обязательный атрибут).
- `filepath`: Путь к JSON файлу ключей (обязательный атрибут).
- `-u`, `--username`: Имя пользователя.
- `-p`, `--password`: Пароль пользователя.
- `-h`, `--help`: Показать это сообщение и выйти.

### Пример использования:

1. Загрузка ключей на панель из JSON:
   ```
   python3 keys.py lj host 192.168.0.1 --filepath path/to/input.json
   ```

2. Загрузка ключей на панель из JSON с указанием имени пользователя и пароля:
   ```
   python3 keys.py lj host 192.168.0.1 --username admin --password 123456 --filepath path/to/input.json
   ```

3. Загрузка ключей на одну панель из JSON файла:
   ```
   python3 keys.py lj host 192.168.0.1 --filepath path/to/input.json
   ```

4. Загрузка ключей на одну панель из JSON файла с указанием имени пользователя и пароля:
   ```
   python3 keys.py lj host 192.168.0.1 --username admin --password secret --filepath path/to/input.json
   ```

5. Загрузка ключей на несколько панелей из CSV файла на основе JSON файла с авторизацией:
   ```
   python3 keys.py lj list path/to/hosts_list.csv --username admin --password secret --filepath path/to/input.json
   ```

6. Загрузка ключей на перечисленные в строке панели на основе JSON файла без авторизации:
   ```
   python3 keys.py lj string "192.168.0.100,192.168.0.101" --filepath path/to/input.json
   ```
   
7. Загрузка ключей на перечисленные в строке панели из JSON файла с авторизацией:
   ```
   python3 keys.py lj string "192.168.0.100,192.168.0.101" --username user --password top_secret --filepath path/to/input.json
   ```

## Команда `zipup`

Загрузить ключи на панель из ZIP-архива

#### Атрибуты
- `путь_к_архиву`: Путь к ZIP-архиву с ключами (обязательный атрибут)

#### Ограничения файла внутри архива
Файл внутри архива должен иметь один из следующих форматов:
- Файл в формате CONF: `<ip_address>.conf` (например, `10.80.1.200.conf`)

    Пример содержимого файла .conf:
    ```
    [KEYS]
    KeyValue1=000000C2137B42
    KeyApartment1=0
    KeyIndex1=0
    KeyValue2=000000C21252E2
    ```

- Файл в формате JSON: `<ip_address>.json` (например, `10.80.1.200.json`)

    Пример содержимого файла .json:
    ```
    "{\"Keys\": [{\"Key\": \"000000034D9502\",\"Apartment\": \"0\"}]}"
    ```

#### Примеры
1. Загрузка ключей на панель из ZIP-архива:
   ```
   python3 keys.py zipup path/to/archive.zip
   ```
   
2. Загрузка ключей на панель из ZIP-архива с указанием пути к архиву:
   ```
   python3 keys.py zipup /absolute/path/to/archive.zip
   ```

## Команда `add`

Команда add используется для загрузки одиночного ключа на панель.

### Подкоманда `host`

Загружает ключ на одну панель. Идеальна для работы с одним устройством, когда нужно восстановить или заменить ключи.

#### Аргументы:

- `ip`: IP адрес панели (обязательный атрибут).
- `u`, `-username`: Имя пользователя, зарегистрированного на панели Beward.
- `p`, `-password`: Пароль пользователя, зарегистрированного на панели Beward.
- `h`, `-help`: Показать это сообщение и выйти.
- `-k`, `--key`: UID ключа, дополненный при необходимости до 7 байт нулями
    
    > (обязательный)
    > 
- `-a`, `--apartment`: Квартира, к которой привязан ключ. Не используется, если ключ не привязывается к квартире
    
    > (необязательный, значение по умолчанию: "0")
    > 
- `-t`, `--type`: Тип электронного ключа/метки:
    
    > (необязательный, значение по умолчанию: `1`)
    > 
    - `0` - Ultralight C
    - `1` - Mifare Classic
    - `2` - Mifare Plus SE
    - `3` - Mifare Plus X
    
    Допустимые значения: `0, 1, 2, 3`
    
- `-pm`, `--protectedmode`: Защищенный режим:
    
    > (необязательный, значение по умолчанию: `0`)
    > 
    - `0` - выключен
    - `1` - включен
- `-ci`, `--cipherindex`: Индекс шифра. От 0 до 65535
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-nce`, `--newcipherenable`: Смена шифра для ключа:
    - `0` - выключена
    - `1` - включена
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-nci`, `--newcipherindex`: Новый индекс шифра. От 1 до 65535
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-c`, `--code`: Код. Формат - 4 байта в hex, например: 01FFAE67.
    
    Значение от 00000001 до FFFFFFFF
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-s`, `--sector`: Номер сектора. Значение от 1 до 9999
    
    > (необязательный, значение по умолчанию: `1`)
    > 
- `-o`, `--owner` : Владелец
    
    > (необязательный, значение по умолчанию: "")
    > 
- `-ap`, `--autopersonalize` : Автоперсонализация:
    - `0` - включена
    - `1` - выключена
    
    > (необязательный, значение по умолчанию: "0")
    > 
- `-sv`, `--service` : Сервисный ключ:
    - `0` - включен
    - `1` - выключен
    
    > (необязательный, значение по умолчанию: "0")
    >

    > (необязательный, значение по умолчанию: `0`)
    > 
- `-nce`, `--newcipherenable`: Смена шифра для ключа:
    - `0` - выключена
    - `1` - включена
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-nci`, `--newcipherindex`: Новый индекс шифра. От 1 до 65535
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-c`, `--code`: Код. Формат - 4 байта в hex, например: 01FFAE67.
    
    Значение от 00000001 до FFFFFFFF
    
    > (необязательный, значение по умолчанию: `0`)
    > 
- `-s`, `--sector`: Номер сектора. Значение от 1 до 9999
    
    > (необязательный, значение по умолчанию: `1`)
    > 
- `-o`, `--owner` : Владелец
    
    > (необязательный, значение по умолчанию: "")
    > 
- `-ap`, `--autopersonalize` : Автоперсонализация:
    - `0` - включена
    - `1` - выключена
    
    > (необязательный, значение по умолчанию: "0")
    > 
- `-sv`, `--service` : Сервисный ключ:
    - `0` - включен
    - `1` - выключен
    
    > (необязательный, значение по умолчанию: "0")
    >

## Команда `rm`

Команда rm используется для удаления ключей с панели. 

### Подкоманда `host`

Удаляет ключи с одной панели. Идеальна для работы с одним устройством, когда нужно удалить ключи.

#### Аргументы:

- `ip`: IP адрес панели (обязательный атрибут).
- `u`, `-username`: Имя пользователя, зарегистрированного на панели Beward.
- `p`, `-password`: Пароль пользователя, зарегистрированного на панели Beward.
- `h`, `-help`: Показать это сообщение и выйти.
- `h`, `-help`: Показать помощь и выйти
- `k`, `-key`: UID ключа, дополненный при необходимости до 7 байт нулями (домофоном оперирует UID ключа в обратном порядке байт)
- `a`, `-apartment`: квартира, к которой привязан ключ
- `i`, `-index`: индекс ключа в базе
- `-all`: удалить все ключи

### Подкоманда `list`

Удаляет ключи с нескольких панелей перечисленных в CSV файле.

#### Аргументы:

- `csvpath`: Путь к CSV файлу со списком адресов (обязательный атрибут).
- `u`, `-username`: Имя пользователя.
- `p`, `-password`: Пароль пользователя.
- `h`, `-help`: Показать это сообщение и выйти.
- `h`, `-help`: Показать помощь и выйти
- `k`, `-key`: UID ключа, дополненный при необходимости до 7 байт нулями (домофоном оперирует UID ключа в обратном порядке байт)
- `a`, `-apartment`: квартира, к которой привязан ключ
- `i`, `-index`: индекс ключа в базе
- `-all`: удалить все ключи

### Подкоманда `string`

Позволяет удалить ключи с панелей, адреса которых указаны как строка с разделителями. Это удобно, если нужно быстро выполнить описанную задачу без необходимости создания внешних файлов.

#### Аргументы:

- `string`: Список IP-адресов, разделенных запятой (обязательный атрибут).
- `u`, `-username`: Имя пользователя.
- `p`, `-password`: Пароль пользователя.
- `h`, `-help`: Показать это сообщение и выйти.

- `k`, `-key`: UID ключа, дополненный при необходимости до 7 байт нулями (домофоном оперирует UID ключа в обратном порядке байт)
- `a`, `-apartment`: квартира, к которой привязан ключ
- `i`, `-index`: индекс ключа в базе
- `-all`: удалить все ключи

## Примечание:

Для использования подкоманды `list`. CSV файл должен быть оформлен по следующему шаблону, где каждая строка в файле представляет собой IP-адрес панели, а столбцы разделены символом `<;>`:

```
IP;Name
10.80.61.196;
10.80.61.197;
10.80.61.198;
10.80.61.206;
```

Столбец `Name` является необязательным и может быть оставлен пустым. Убедитесь, что файл соответствует указанному формату для корректной обработки подкомандой `list`.
